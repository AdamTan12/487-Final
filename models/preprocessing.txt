# Classifier preprocessing contract.
#
# Source of truth for the values both the Python training pipeline and
# the C++ inference path must use. Any drift here silently degrades
# accuracy — update both sides in lockstep.

input_size:  224x224
color_order: BGR
mean:        [0.485, 0.456, 0.406]
std:         [0.229, 0.224, 0.225]
scale:       0.00392156862745098    # 1.0 / 255.0
class_order: [like, dislike, peace, mute, stop, gun, ok, fist, palm]

notes: |
  These values must be replicated EXACTLY in both training (Python) and
  inference (C++). Any mismatch causes major accuracy drop.

  Channel order is BGR (no swap) because cv::dnn::blobFromImage in the
  C++ pipeline does not swap by default and the Python pipeline reads
  via cv2.imread (also BGR). If you ever switch to PIL on the Python
  side, you MUST insert a BGR<->RGB swap to keep this contract.

  The bbox crop is padded by ClassicalDetectorConfig.padding_ratio
  (default 0.15) before resizing, and clamped to frame bounds.
